## Methodology

### Evaluation Dataset
- **Gold standard**: 50 hand-curated Q&A pairs (30 standard, 8 out-of-scope, 6 ambiguous, 6 multi-hop).
- **Synthetic extension**: [N] pairs generated by Llama 3.2 1B from document chunks. Used for qualitative inspection only; not included in primary metrics due to lexical bias.
- **Evaluation scope**: Retrieval metrics computed on standard questions only (n=30); refusal metrics on adversarial subset (n=14).

### Metrics
- **Primary retrieval metric**: nDCG@5 (balances precision and rank position).
- **Supporting**: MRR, Precision@5, Recall@5, Hit Rate@5.
- **Generation**: Faithfulness heuristic (keyword overlap proxy), citation rate, refusal accuracy.
- **End-to-end (finalist configs only)**: RAGAS Faithfulness, AnswerRelevancy, ContextPrecision.
- **Statistical reporting**: Bootstrap percentile CI (n=1000, α=0.05); Wilcoxon signed-rank test for pairwise comparisons.

### Reproducibility
All experiments seeded at 42. Non-deterministic components: Ollama LLM (CPU BLAS thread ordering), BM25 tokenisation (deterministic). To reproduce:
```bash
python experiments/embedding_experiments/run_embedding_comparison.py --config experiments/config/embeddings.yaml
python experiments/retrieval_experiments/run_retrieval_ablation.py --config experiments/config/retrieval.yaml
python experiments/prompt_experiments/run_prompt_ablation.py --config experiments/config/prompts.yaml
```

## Limitations

1. **Small eval set (n=50)**: Bootstrap CIs are wide; improvements < 0.05 nDCG should not be treated as reliable signals. Collect more gold pairs before the next evaluation round.

2. **Single-domain corpus**: The ACME KB is a clean, well-structured document. Results may not transfer to noisy, multi-document corpora.

3. **Judge == generator bias**: All RAGAS LLM-as-judge scores use Llama 3.2 1B as both generator and judge. A stronger judge (e.g., Llama 3.1 70B via API) would give more reliable absolute scores.

4. **CPU-only inference**: Latency numbers are for CPU-only execution. On GPU, the relative ranking of methods may shift (cross-encoder reranking becomes relatively cheaper).

5. **Synthetic QA bias**: Synthetic questions generated from chunks have higher overlap with the source chunk vocabulary, inflating recall for all retrieval methods. Excluded from primary results.

6. **Relevance labelling**: Pseudo-relevance labels derived from keyword overlap with gold answers. True relevance requires human annotation, especially for multi-hop questions.

## Future Work

1. **Expand gold dataset**: Aim for n=200 with diverse phrasings; use crowdsourcing or stronger LLM for generation + human review.
2. **Re-ranking with a stronger model**: Try `cross-encoder/ms-marco-MiniLM-L-12-v2` (larger) or `BAAI/bge-reranker-base`.
3. **Fine-tune embedding model**: Few-shot domain adaptation via TSDAE or domain-specific contrastive learning on (query, positive, negative) triples.
4. **Caching layer**: For repeated queries, an exact-match cache eliminates retrieval latency entirely.
5. **Multi-vector retrieval (ColBERT)**: Late-interaction models may outperform bi-encoder retrieval on this corpus; requires `ragatouille` library.
6. **GPU evaluation**: Re-run timing benchmarks on GPU to update the Pareto frontier for deployment decisions.
